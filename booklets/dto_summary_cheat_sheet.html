Hop From → To Protocol DTO Fields
1 Simulator → Broker HTTP GET SensorSummary[] id, name, category, region, coordinates, sampling_rate_hz, websocket_url
2 Simulator → Broker WebSocket SensorMeasurement timestamp, value — no sensorId (implicit from WS path)
3 Broker → Replica WebSocket BrokerMessage sensorId, timestamp, value — our DTO, adds sensorId
4 Simulator → Replica SSE ShutdownCommand command: "SHUTDOWN" — kills exactly one replica
5 Replica → PostgreSQL JDBC DetectedEvent sensor_id, event_type, dominant_freq_hz, magnitude, window_start, window_end, region, replica_id
6 Replica → Dashboard REST / WS EventResponse id, sensorId, sensorName, eventType, dominantFreqHz, magnitude, detectedAt, region, category
Dedup key: The unique constraint (sensor_id, event_type, window_start) on the database ensures that when all 3 replicas detect the same earthquake from the same sensor in the same time window, only one row is stored.
